About KeyProgrammer Parser

Supported devices

The parser works with device full file system extractions made by your favorite extraction tools. Supported device families include:

  • Autel: KM100, IM508, MX808 (MaxiIM/MaxiSys style exports)
  • XTool: D7 and related AutoPad/AutoPadPro exports
  • Other devices using similar file structures may also work

How to use

  1. Choose Input Type: "Zip File" or "Folder".
  2. Browse and select the input file or folder containing the device extraction.
  3. Select a folder where the HTML report should be saved (choose a drive with free space because a temp folder may also be placed here).
  4. Click "Start Parsing" — progress appears in the bar and a short status message is shown. When finished the report preview will load and an "Open report" button will become available.

What the report contains

The generated HTML report summarizes the key findings so you can quickly review evidence:

  • Extracted VINs and any matching vehicle metadata
  • Discovered Wi‑Fi networks and saved credentials (when present)
  • Detected key/key‑programming records and transponder information
  • Relevant log excerpts, timestamps and source file locations

Quick troubleshooting

  • If the report does not appear: check the final status line for errors and confirm the chosen report folder exists and has free space.
  • If extraction fails with "No space left on device": pick a report folder on a different drive with more free space and try again.
  • If the app reports files locked during cleanup, the report will still be written — try opening it from the report folder or use the "Open report" button in the app.
  • If the parser finds unexpected or incomplete results, attach the generated HTML report and a short description when requesting support (see contact details below).

Privacy & file handling

All processing is local. The app writes a single HTML report into the folder you choose. If you are online, VIN look ups will be performed through an API with NHTSA. Temporary extraction folders are removed when possible; if a temp folder remains it can be safely deleted later.
